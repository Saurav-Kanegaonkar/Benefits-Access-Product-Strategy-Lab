# Data Dictionary

| Table | Grain | Purpose |
|---|---|---|
| `product_workflows.csv` | Product workflow | Product area, persona, problem, hypothesis, owner, horizon, strategic fit, effort, and risk sensitivity |
| `prd_requirements.csv` | Requirement | Evidence-backed user needs, acceptance criteria, instrumentation, confidence, request volume, severity, and PRD status |
| `weekly_operating_metrics.csv` | Workflow x week | Intake volume, qualification rate, match acceptance, firm response time, client trust, case readiness, and data quality |
| `stakeholder_events.csv` | Evidence event | User friction, firm feedback, support escalation, data gap, policy risk, and estimated impact |
| `experiment_plans.csv` | Workflow experiment | Hypothesis, primary metric, guardrail metric, sample plan, detectable lift, and decision rule |
| `public_benchmarks.csv` | Public benchmark | Source, metric, benchmark value, domain, and intended use in product strategy |
| `analysis/outputs/roadmap_queue.csv` | Scored workflow | Roadmap prioritization, launch status, next decision, request count, modeled impact, and readiness |
| `analysis/outputs/prd_cards.csv` | PRD slice | Selected PRD problem, user story, acceptance criteria, instrumentation, and validation plan |
| `analysis/outputs/readiness_register.csv` | Workflow launch gate | Launch status, readiness score, weakest gate, owner, and next decision |
| `analysis/outputs/experiment_readout.csv` | Experiment readout | Experiment design and launch measurement plan aligned to each roadmap candidate |
| `analysis/outputs/platform_readiness.csv` | Workflow data contract | Source domain, required fields, freshness SLA, missingness, model action, public benchmark, confidence, and launch blocker |
