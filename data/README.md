# Data Sources

Synthetic source-style data for a benefits access and legal-help product strategy workbench.

- `product_workflows.csv`: product workflow records across intake, lawyer matching, evidence readiness, marketplace health, and case collaboration.
- `prd_requirements.csv`: PRD-ready requirements with persona, evidence source, severity, request volume, user need, and instrumentation.
- `weekly_operating_metrics.csv`: weekly product operating metrics by workflow.
- `stakeholder_events.csv`: stakeholder evidence events, including user friction, firm feedback, data gaps, policy risk, support escalation, and experiment ideas.
- `experiment_plans.csv`: experiment plans with primary metric, guardrail metric, sample plan, minimum detectable lift, and decision rule.

The datasets are deterministic synthetic data generated by `scripts/score_operating_data.py`. They do not represent real client, claim, law-firm, benefits, legal, agency, insurer, or company performance.
